# P15 学习闭环完成报告

## 目标

P15 的目标不是增加一个好看的页面，而是让工具越用越准。

这次实现的是可解释规则学习：用户在客户详情里标记触达、测试、成交、无效后，系统会把反馈沉淀到平台、客户类型、来源和关键词上。

## 已完成

1. 关键词学习

- 成交、测试、可推进率高的关键词会涨权。
- 无效多、只产噪音、没有形成客户的关键词会降权。
- 低质量关键词在满足明确条件后会被暂停。
- 每个关键词都有调权原因，不做黑盒判断。

2. 来源学习

- 来源会根据质量分、反馈分、平台学习结果、连续失败和实际产出计算学习优先级。
- 高优先级来源标记为 `boost`。
- 低优先级且已有足够失败证据的来源标记为 `cooldown`，可自动暂停。

3. 平台和客户类型学习

- 平台、客户类型继续按触达、测试、成交、无效计算权重。
- 反馈不足时保持克制，不因为少量样本大幅改变策略。

4. 客户分数微调

- 应用学习后，客户优先级会根据平台和客户类型反馈微调。
- 已成交、已测试会加分；无效会明显降分。
- 调整会写入 `next_action`，保留“学习权重”说明。
- 学习调权是幂等的：反复点击不会把客户分数一遍遍累加。

5. 转化反馈页升级

- `/feedback` 新增学习预览。
- 页面展示关键词学习、来源学习、平台权重、客户类型权重。
- 新增“应用学习调权”按钮。
- `/feedback.csv` 现在导出反馈汇总、来源学习、关键词学习。

## 当前本地数据回放

本地测试时，当前有效关键词样本较少，主要是“防关联”。

学习预览判断为：

- 关键词：防关联
- 样本：3 条线索 / 3 个客户
- 当前权重：35
- 建议：保持
- 原因：反馈不足，先保持

这说明 P15 没有为了“看起来有变化”而乱调权。等后续用户开始标记“无效、已测试、成交”后，关键词和来源权重会明显分化。

执行一次真实调权后，本地结果：

- 客户微调：16
- 来源调权：246
- 关键词调权：0
- 暂停关键词：0

关键词没有变化是因为当前没有足够强的正负反馈，这是合理状态。

## 验收结果

已满足：

- 用户标记无效后，同类关键词和来源会进入降权逻辑。
- 用户标记测试/成交后，同类关键词、平台、客户类型会进入涨权逻辑。
- 来源效果可以按学习优先级、反馈分、质量分和原因排序。
- 所有学习结果都有原因说明。
- 不使用复杂机器学习黑盒。
