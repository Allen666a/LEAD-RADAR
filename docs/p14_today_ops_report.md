# P14 今日作战台完成报告

完成时间：2026-06-13

## 目标

P14 的目标是让用户每天打开工具后，30 秒内知道今天先做什么。

不做花哨大屏，不堆无意义总数，只保留能直接推动获客的动作。

## 已完成

### 1. 首页作战台升级

首页 `/` 新增“今天先做这 3 件事”。

每条建议包含：

- 优先级
- 数量
- 为什么要做
- 下一步怎么做
- 直接处理入口

当前会根据实际数据生成这些动作：

- 到期复访
- 联系已有触达入口的高分客户
- 补齐高价值线索联系方式
- 推进测试包和测试后复访
- 人工审核风险线索
- 处理受阻平台
- 暂停低质或受阻来源
- 放大高质量关键词
- 暂停无产出关键词
- 补国内平台新鲜线索

### 2. 今日页合并为同一作战台

`/today` 不再使用旧的简单列表，而是复用 P14 作战台。

这样不会出现：

- 首页一套逻辑
- 今日跟进另一套旧 UI
- 用户不知道哪个页面才是主入口

### 3. 采集取舍区

首页新增“今天的采集取舍”：

- 该放大的关键词
- 该暂停或改写的关键词
- 来源警报

这对应 P14 验收里的：

- 哪些关键词带来高质量线索
- 哪些关键词应该暂停
- 哪些来源质量变差

### 4. 数据层增强

`P5Workbench` 新增：

- `p14_actions`
- `high_keywords`
- `pause_keywords`
- `source_warnings`

新增 `P14Action`，用于承载首页行动建议。

### 5. 现有销售任务继续保留

原来的今日私聊队列继续保留：

- 补联系方式
- 首触达
- 到期复访
- 发测试包
- 测试后跟进
- 人工审核

P14 不是替代销售队列，而是在销售队列上方告诉用户“先做哪 3 件事”。

## 业务价值

P14 解决的是“工具很多按钮，但不知道今天干什么”的问题。

现在主流程变成：

```text
打开首页
看今天先做 3 件事
进入补联系方式 / 平台采集 / 来源管理 / 今日私聊队列
处理完后记录反馈
```

这比继续加平台或加按钮更实用。

## 明确不做

- 不做营销大屏。
- 不做无行动价值的总览数字。
- 不把所有模块塞进首页。
- 不自动触达客户。
- 不绕过平台风控。

进入 P15：学习闭环。

重点是让用户的反馈真正影响系统：

- 标记无效后，同类垃圾减少。

- 来源、关键词、客户类型能按真实转化表现排序。
