# P11 线索质量引擎完成报告

日期：2026-06-12

## 目标

P11 的目标不是增加功能数量，而是提高线索质量：

- 高优先线索必须是 2026 年后的新线索。
- 高优先线索必须和动态住宅 IP 或明确代理/IP 痛点强相关。
- 教程、新闻、招聘、官网软文、同行广告不能进入高优先。
- 每条线索必须能解释为什么值得看。

## 已完成

### 1. 新增 P11 质量闸门

新增文件：

```text
app/services/p11_quality_gate.py
```

核心能力：

- 判断原文时间是否满足 2026+。
- 判断目标客户画像。
- 判断动态住宅 IP 匹配度。
- 判断痛点和购买意图。
- 判断教程、新闻、招聘、官网软文、同行广告等噪音。
- 输出 P11 等级、分数、画像、匹配理由、拒绝原因和下一步动作。

### 2. 入库链路接入 P11

修改文件：

```text
app/services/ingest.py
```

现在新采集数据必须同时满足：

```text
通过时效门槛
通过原有评分
通过 P11 质量闸门
通过公开平台准入规则
```

不再允许“关键词命中但质量很差”的内容进入主池。

### 3. 详情页证据接入 P11

修改文件：

```text
app/services/lead_evidence.py
```

线索详情页现在会显示：

- P11质量
- 客户画像
- 动态住宅匹配
- 原文时间
- 痛点证据
- 拒绝/风险原因
- 下一步动作

## 剩余问题

P11 已完成，但还不是最终产品状态。

下一阶段需要进入 P12：

```text
国内平台采集稳定化
后台采集进度可视化
```
