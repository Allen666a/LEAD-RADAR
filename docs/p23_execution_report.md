# P23 执行报告：线索列表产品化收敛

## 目标

把 `/leads` 从“结果展示页”升级成“线索初筛工作台”。用户不用每条都点详情，也能在列表里判断是否值得看、是否过旧、是否缺联系方式，并直接做下一步动作。

## 已完成

1. 线索行新增初筛字段
   - 在 `app/services/lead_finder.py` 扩展 `LeadFinderRow`。
   - 新增字段：
     - `screening_label`
     - `screening_tone`
     - `time_label`
     - `time_tone`
     - `action_label`
     - `action_kind`
     - `prospect_id`

2. 列表显示更明确
   - 每条线索展示筛选结论：
     - 优先看
     - 场景匹配
     - 可观察
     - 先复核
     - 旧线索
   - 每条线索展示时间判断：
     - 2026 新线索
     - 时间未知
     - 旧线索

3. 新增列表内动作
   - 新增 `POST /leads/{mention_id}/action`。
   - 支持：
     - 补联系方式
     - 加入跟进
     - 标记无效
   - 操作后回到当前筛选条件。

4. 动作会写回数据
   - 补联系方式：线索变为已查看，客户变为已筛选，并写状态事件。
   - 加入跟进：线索变为已查看，客户变为已筛选，并写状态事件。
   - 无效：线索和客户都标记无效，客户进入 suppressed，并写结果事件。

5. 空状态优化
   - 没有符合条件线索时，不再只显示一句空提示。
   - 提供动作：放宽条件、检查平台登录。

6. UI 适配
   - 新增初筛标签样式。
   - 动作按钮支持换行。
   - 桌面端无横向溢出。
