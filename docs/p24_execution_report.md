# P24 执行报告：联系方式补全工作台产品化

## 目标

把“补联系方式”从一个泛泛的处理页，升级成真正能辅助人工获客的工作台：快速记录微信、QQ、Telegram、邮箱、官网链接，标记联系方式来源，并且在找不到联系方式时记录明确原因，避免重复无效查找。

## 已完成

1. 快速录入联系方式
   - 支持微信、QQ、Telegram、邮箱、主页/官网独立录入。
   - QQ 不再混在备注里，作为独立联系方式保存。
   - 保存后会更新客户状态为可联系，并写入联系记录。

2. 联系方式来源标记
   - 增加来源选项：主页/个人资料、原文、站内搜索、同名全网搜索、人工确认。
   - 保存联系方式时同步写入来源说明，后续复盘能知道线索是怎么找到的。

3. 找不到联系方式闭环
   - 新增“标记找不到”表单。
   - 支持原因：公开页面未找到、只看到私信入口、页面太泛无法确认本人、意向不够强、平台登录/验证受阻。
   - 标记后会写入下一步动作和触达活动，避免同一条线索反复浪费时间。

4. 页面收敛
   - 联系方式录入区域改成紧凑栅格。
   - 找不到联系方式从普通按钮改成明确的原因选择。
   - 页面在桌面和移动端均无横向溢出。

## 当前限制

- P24 解决的是“补联系方式”的人工工作台，不是自动破解私域联系方式。
- 国内平台仍然只做公开页面和登录会话下可见内容，不做私信、关注、评论等高风险动作。
- 默认筛选下如果没有待补联系方式任务，页面会显示空列表；全量筛选模式已验证正常。
