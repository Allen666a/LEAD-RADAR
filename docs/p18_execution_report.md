# P18 执行报告：待联系工作台产品化

日期：2026-06-13

## 目标

把“待联系”做成真正能干活的补联系方式工作台：用户不需要理解一堆内部指标，只需要知道谁优先补、为什么补、去哪里找、补到后怎么保存并推进。

## 已完成

1. 重做 `/contact-workbench` 页面
   - 改为“任务流”布局，不再是复杂后台表格。
   - 页面核心路径：看优先级 -> 打开补全入口 -> 保存并推进。
   - 每个客户卡片分成三块：优先级、线索判断、联系方式保存面板。
   - 默认空状态增加“去找线索”按钮，避免用户看到空页不知道下一步。

2. 强化补联系方式信息结构
   - 新增联系方式摘要。
   - 新增缺失渠道：微信、QQ、Telegram、邮箱、手机。
   - 新增来源标签，卡片里直接显示这条任务来自哪个来源。
   - 保留原文、主页、百度、Bing、站内搜索、同名搜索等补全入口。

3. 保存和推进动作更清楚
   - 右侧固定显示联系方式表单。
   - 保存后动作是“保存并进入跟进”。
   - 保留可跟进、3 天复查、暂无联系、无效等实际销售动作。
   - 自动补联系方式入口放到页面顶部，支持小批量公开页面补全。

4. 修复 CSV 导出字段错位
   - 补充 `contact_confidence` 和 `contact_records` 表头。
   - 避免导出后 `product_fit` 和联系方式指标错列。

## 当前状态

P18 已完成。这个页面现在更像销售工作台，而不是数据后台。

默认筛选下当前没有“国内待补联系方式”任务，这是数据状态，不是页面错误。切到“全部客户 / 全部平台 / 最低分 0”可以看到现有任务。

## 下一步

P19 建议做“今日作战”产品化：每天只给 3 到 5 个最该做的动作，把找线索、补联系方式、首次触达、复查和无效反馈串起来。这样用户不需要每天在多个页面之间猜下一步。
