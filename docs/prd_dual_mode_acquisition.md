# PRD：Lead Radar 双模式获客系统

## 1. 产品定位

Lead Radar 是面向海外动态住宅 IP 代理业务的垂直获客系统。

它不做通用爬虫，也不做群发营销工具。核心目标是帮助销售找到两类机会：

- 正在表达代理 IP 痛点的人或团队。
- 有长期动态住宅 IP 消耗能力的公司或组织。

产品采用“一套系统，两条流水线”：

```text
需求雷达：发现即时痛点和散落信号。
B2B 客户库：沉淀公司级高价值客户。
```

## 2. 用户和使用场景

### 2.1 主要用户

```text
业务负责人：看今天获客质量和来源效果。
销售：每天处理高价值客户、补联系方式、跟进转化。
运营：维护关键词、来源、采集任务、质检结果。
```

### 2.2 每日核心流程

```text
打开作战台
查看今天可运行来源
查看需求雷达新增痛点
查看 B2B 客户库新增公司
处理高优先级客户
补联系方式
进入今日跟进
记录触达结果
查看来源反馈
```

### 3.2 页面职责

| 页面 | 职责 | 不应该做 |
| --- | --- | --- |
| 作战台 | 双模式总控，告诉用户今天该跑什么、修什么、跟谁 | 不堆所有原始数据 |
| 需求雷达 | 展示社区/问答/社媒/论坛等痛点信号 | 不作为最终客户库 |
| B2B 客户库 | 展示公司级客户画像和意图信号 | 不展示零散帖子为主 |
| 补联系方式 | 找公开商务触达入口并保存证据 | 不做自动骚扰 |
| 跟进池 | 管理销售阶段和下一步动作 | 不承载原始采集噪音 |
| 销售节奏 | 安排复访、触达、复查 | 不自动群发 |
| 来源 | 管理采集源质量和启停 | 不让低质来源无限灌数据 |

## 4. 核心对象

```text
Signal：原始信号。
Prospect：可跟进线索。
Company/Account：公司级客户。
Contact：联系方式。
Activity：跟进活动。
```

对象流转：

```text
Signal -> Prospect -> Company/Account -> Contact -> Activity
```

## 5. 需求雷达

### 5.1 页面目标

发现“大海捞针”中的高相关痛点。

页面回答：

```text
哪里有人现在正在为动态住宅 IP、防关联、403、爬虫、矩阵账号痛？
```

- 知乎
- 百度贴吧
- 小红书
- 抖音
- B 站
- 微博
- V2EX
- SegmentFault
- GitHub issues / discussions
- 跨境卖家论坛
- 私域导入

### 5.3 列表字段


### 5.4 筛选

```text

痛点类型
分数区间
是否可触达
是否风险
是否已转入客户库

```

### 5.5 操作

```text
转为线索
合并到已有客户
标记噪音
标记同行/广告
标记合规风险
加入补联系方式队列
查看原文
```

### 5.6 验收标准

- 每条数据必须显示来源平台。
- 高分线索必须说明为什么和动态住宅 IP 相关。
- 新手教程、新闻、同行软文不能进入高优先级。
- 风险线索不能直接进入触达。
- 需求雷达中可以一键转 Prospect 或合并到 Company。

## 6. B2B 客户库

### 6.1 页面目标

沉淀有长期代理 IP 消耗能力的公司级客户。

页面回答：

```text
哪些公司值得销售认真跟？
```

### 6.2 数据来源

P0/P1 优先：

- 已有 Prospect 合并
- GitHub 组织/项目
- 公司官网关键词
- 手动导入

P2 扩展：

- 招聘信号
- SaaS/工具目录
- 跨境服务商目录
- 数据采集/广告验证/价格监控公司目录

### 6.7 验收标准

- B2B 客户库第一列必须是公司/官网，而不是帖子标题。
- 每个高优先级公司必须有明确代理需求理由。
- 每个公司必须能看到信号来源。
- 公司级去重必须避免同一域名重复出现。
- 可导出销售可用 CSV。

## 7. 作战台

### 7.1 页面目标

每天第一屏，直接回答：

```text
今天该跑什么？
今天该修什么？
今天该跟谁？
```

### 7.2 模块

```text
双模式统计
需求雷达今日新增
B2B 今日新增
可运行来源
受阻来源
高优先级客户
待补联系方式
今日跟进任务
来源质量提醒
```

### 7.3 验收标准

- 不能只显示按钮和总数。
- 必须给出可执行建议。
- 必须区分需求雷达和 B2B。
- 必须显示受阻平台和下一步处理。

## 8. 补联系方式

### 8.1 页面目标

把高价值客户变成可触达客户。

### 8.2 支持入口

```text
Company
Prospect
Signal
```

### 8.3 字段

```text
联系方式类型
值
来源 URL
置信度
是否商务联系方式
是否个人数据
备注
保存时间
```

### 8.4 验收标准

- 保存联系方式必须带来源或备注。
- 自动补全结果必须有置信度。
- 风险联系方式不能直接进入触达。
- 同行/广告/无效客户不进入补全队列。

### 9.3 验收标准

- 每个 A 类客户必须有下一步动作。
- 每次状态变化必须有 Activity。
- 有退订/拒绝/投诉必须进入抑制名单。
- 不做自动群发，触达前保留人工确认。

## 10. 合规要求

必须遵守：

- 优先公司级数据。
- 优先公开商务联系方式。
- 不绕验证码。
- 不绕登录限制。
- 不批量私信。
- 不自动群发。
- 不采集敏感个人数据。
- 保留来源证据。
- 支持 do-not-contact。

## 11. P0 范围

P0 必须完成：

```text

B2B 客户库页面
作战台双模式改造
Company/Signal/Contact/Activity 最小对象
四分制评分
公司级去重
B2B CSV 导出
基础合规字段
```

P0 不做：

```text
自动群发
复杂 CRM 集成
大规模第三方数据购买
绕过平台风控
全量平台采集
```

## 12. 成功指标

```text
需求雷达高分线索相关率 >= 70%
B2B 每批产出 10 个有明确需求理由的公司
高价值客户联系方式补全率 >= 30%
A 类客户下一步动作覆盖率 >= 90%
来源噪音率持续下降
销售能用作战台完成一天工作安排
```
