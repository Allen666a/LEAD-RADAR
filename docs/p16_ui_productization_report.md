# P16 UI 产品化收口完成报告

## 目标

P16 的目标不是继续加按钮，而是把已经做出来的获客能力收成一个每天能用的工作台。

这次优化围绕一条真实使用链路：

```text
平台采集 -> 线索池 -> 国内质检 -> 补联系方式 -> 跟进池 -> 学习反馈
```

## 已完成

1. 导航重构

- 左侧菜单从“很多名词入口”收敛为核心工作链路。
- 高频入口保留在主导航：今日作战、线索池、平台采集、国内质检、补联系方式、跟进池、客户画像、学习反馈。
- 低频入口收进“更多工具”：平台状态、来源管理、效果分析、策略词库、日报、设置等。
- 保留当前模块选中态。
- 保留左侧菜单滚动位置，避免每次点击后跳到最上面。

2. 顶部工作流条

- 所有核心页面顶部都有统一工作流：
  - 采集
  - 发现
  - 质检
  - 补联
  - 跟进
- 用户不用猜下一个页面在哪里。

3. 后台采集状态

- 顶部增加采集状态入口。
- 会读取 `/session-collector/progress.json`。
- 可以显示：
  - 采集空闲
  - 某个平台正在采集中
  - 某个平台需要处理
  - 上次入库数量

4. 表格体验收口

- 宽表限制在自己的容器内，不再撑爆整个页面。
- 自动生成顶部横向滚动条，用户不用拉到页面最底部才能横向滚动。
- 表头在桌面端粘住，长表阅读时不容易迷失列名。
- 移动端禁用过重的粘性效果，避免小屏错位。

5. 视觉统一

- 统一了页面背景、卡片、按钮、导航、状态徽标和工作流入口。
- 页面主标题和顶部动作收口，减少分散按钮。
- 质检、客户、作战台继续使用卡片式队列，减少宽表依赖。

## 后续建议

下一阶段不建议继续泛泛加功能。

优先做真实运营验证：

1. 每天只跑 2-3 个最稳平台。
2. 人工质检 30 条线索。
3. 标记无效、已触达、已测试、成交。
4. 观察 P15 学习闭环是否把关键词和来源越调越准。
5. 根据真实反馈再决定扩平台还是加联系方式能力。
