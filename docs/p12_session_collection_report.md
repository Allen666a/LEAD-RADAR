# P12 国内平台采集稳定化完成报告

完成时间：2026-06-12

## 目标

P12 不继续盲目扩平台，先把“登录后后台采集”做成可观察、可诊断、可恢复的实用能力。

重点解决：

- 点击后台采集后，用户能看到是否已排队、是否运行、当前平台和关键词。
- 平台失败时不再只显示含糊错误，而是显示标准失败原因。
- 失败后给出下一步动作，避免用户反复乱点导致账号风险升高。
- 继续遵守边界：不破解验证码、不绕登录、不私信、不关注、不评论、不做高频批量互动。

## 已完成

### 1. 采集进度结构升级

`SessionProgress` 新增 `failure_code` 字段。

后台采集在以下场景会写入失败码：

- 缺少 Playwright 依赖
- 平台暂不支持
- 登录态未就绪
- 平台验证码/风控暂停
- 浏览器 profile 被登录窗口占用
- 采集器异常
- 质量过滤
- 页面无候选

### 2. 标准失败原因

新增/补齐标准诊断：

- 未登录
- 登录失效
- 验证码/安全验证
- 访问频率过高
- 页面结构变化
- 页面无候选
- 平台拦截
- 账号异常
- 时间不合格
- 质量不合格
- 采集依赖缺失
- 数据库会话缺失
- 浏览器 profile 被占用

每个失败原因包含：

- 可读标签
- 原因解释
- 下一步建议

### 3. 进度 API 升级

`/session-collector/progress.json` 现在返回：

- `state`
- `state_label`
- `platform`
- `platform_label`
- `keyword`
- `fetched`
- `inserted`
- `candidates`
- `rejected`
- `high_quality`
- `failure_code`
- `failure_label`
- `failure_reason`
- `next_step`
- `running`
- `stale`

如果后台任务超过 15 分钟没有进度更新，会自动标记为 `stale`，并提示用户刷新状态或重启服务后单平台试跑。

### 4. 页面可视化

`/session-collector` 进度卡新增：

- 当前阶段
- 当前平台
- 当前关键词
- 读取数
- 入库数
- 候选/过滤数
- 高质量数
- 失败原因
- 失败解释
- 下一步动作

点击“后台采集”后会立即显示“已排队/正在启动”，随后自动轮询进度。

### 5. 避免历史失败误导

修正了一个关键细节：

任务处于 `queued / starting / running` 时，不再把历史失败原因强行带到当前进度里。只有当前任务失败或任务停滞时，才显示失败诊断。

## 剩余风险

国内平台仍然有强风控，P12 不解决验证码破解、绕登录和高频采集，这些也不应该做。

后续如果某个平台长期显示：

- 页面无候选
- 页面结构变化
- 质量过滤

就应该进入平台适配器修复或关键词策略调整，而不是继续加按钮。
