# P20 执行报告：结果反馈闭环产品化

## 目标

把结果反馈从“看统计”升级成“记录真实结果、沉淀无效原因、反哺来源和关键词权重”的工作台。原则是只保留会影响获客动作的能力，不做鸡肋按钮。

## 已完成

1. 新增结构化反馈服务 `app/services/feedback_ops.py`
   - 支持结果：已触达、已发测试、继续跟进、成交、无效。
   - 支持原因：测试效果好、需求紧急、缺联系方式、价格不合适、其实要静态 IP、线索过旧、同行广告、高风险用途、不是目标客户等。
   - 记录后自动更新客户状态、复访时间、下一步动作、反馈事件。

2. 新增反馈提交入口
   - 新增 `POST /feedback/record`。
   - 单条反馈只执行轻量学习调权，不触发全量客户画像重建，避免导入/临时客户被重建流程覆盖。
   - “应用学习调权”按钮仍保留全量学习能力。

3. 重做 `/feedback` 页面
   - 顶部显示核心指标：客户画像、已推进、已发测试、已成交、反馈覆盖。
   - 主区显示“快速记录反馈”候选线索。
   - 侧栏显示反馈原因分布、当前缺口、最近反馈。
   - 下方保留学习建议和关键词学习预览，但减少表格密度。

4. CSV 增强
   - `/feedback.csv` 新增反馈原因分布导出，方便后续复盘哪些原因导致无效或成交。

5. UI 适配
   - 新增反馈页专用 CSS。
   - 桌面无横向溢出。
   - 窄屏自动改为单列布局。

## 当前判断

P20 已完成。现在工具的反馈闭环开始具备产品级雏形：不是只展示数据，而是能把人工销售结果变成下一轮采集、筛选、跟进的输入。

## 下一步建议：P21

P21 建议做“新手式工作流入口”：用户进来只看到三个动作。

1. 找新线索
2. 补联系方式
3. 记录结果

所有高级模块继续保留，但默认隐藏在高级工具里。目标是让非技术用户也能不看说明直接用起来。
