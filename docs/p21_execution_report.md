# P21 执行报告：三步式新手工作台

## 目标

把首页从“功能堆叠页”改成傻瓜式入口。用户进来不需要理解一堆模块名，只按三步走：

1. 找新线索
2. 补联系方式
3. 记录结果

高级功能继续保留，但不再抢首页主入口。

## 已完成

1. 新增首页服务层 `app/services/home_ops.py`
   - 聚合首页所需的少量关键数据。
   - 自动计算当前最该做的一步。
   - 只暴露三类动作：找线索、补联系方式、记录结果。

2. 替换 `/` 首页
   - 首页不再复用 `p5_workbench.html`。
   - 新增 `app/templates/home.html`。
   - `/p5-workbench` 仍保留为高级任务页。

3. 新首页结构
   - 顶部：一句话说明今天怎么获客。
   - 中部：三张主流程卡片。
   - 状态：2026 年线索、高分线索、今天可处理、平台需处理。
   - 下方：最多展示 5 条今天优先目标。
   - 右侧：明确告诉用户日常只用三个入口。

4. UI 优化
   - 新增首页专用 CSS。
   - 桌面端无横向溢出。
   - 小屏自动变单列。
   - 文案统一中文，减少后台感和术语堆叠。

## 测试结果

1. Python 编译通过
   - `app/main.py`
   - `app/services/home_ops.py`

2. HTTP 路由通过
   - `GET /` 返回 200。
   - `GET /p5-workbench` 返回 200。
   - `GET /leads` 返回 200。
   - `GET /feedback` 返回 200。

3. 浏览器截图通过
   - 截图：`data/p21-home-workflow.png`
   - 检查结果：无横向溢出，首页标题正常，3 个主步骤正常展示，优先目标正常展示。

## 当前判断

P21 完成后，工具入口更接近普通用户能直接使用的产品：不是让用户先理解所有模块，而是直接告诉他今天该做什么。

## 下一步建议：P22

建议继续做“首屏任务引导和空状态治理”：

1. 如果没有线索，首页直接引导去登录平台或启动采集。
2. 如果有线索但无联系方式，首页强调补联系方式。
3. 如果有触达但没反馈，首页强调记录结果。
4. 所有空状态都给明确按钮，不显示空表格。
