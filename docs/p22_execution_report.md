# P22 执行报告：空状态和首屏引导治理

## 目标

让工具在“没有数据、平台受阻、缺联系方式、缺反馈”这些情况下也能直接告诉用户下一步做什么。重点不是增加按钮，而是减少用户判断成本。

## 已完成

1. 首页状态判断增强
   - 在 `app/services/home_ops.py` 增加 `HomeGuidance` 和 `HomePlatformAlert`。
   - 首页会根据当前数据自动生成行动建议。
   - 优先判断：无新线索、平台受阻、待补联系方式、待反馈、今日无任务。

2. 首屏行动建议
   - 首页新增“当前建议”卡片。
   - 每张卡片都有明确按钮：
     - 去找线索
     - 查看平台状态
     - 去补联系方式
     - 记录结果
     - 放宽条件

3. 空状态治理
   - “暂无今日优先目标”不再只显示一句空提示。
   - 现在会给出两个动作：去找线索、放宽条件。

4. 平台受阻提示
   - 首页新增“平台需要处理”区域。
   - 展示平台、问题类型、下一步处理建议。
   - 引导到平台登录/会话采集页处理登录、验证、频控、低质过滤等问题。

5. UI 适配
   - 新增建议卡、平台提醒、空状态按钮样式。
   - 桌面和移动端均无横向溢出。

## 当前判断

P22 完成后，工具不再只是展示状态，而是开始能“带着用户走”。当某一步没数据或受阻时，页面会给出下一步动作。

## 下一步建议：P23

建议做“线索列表产品化收敛”：

1. 把 `/leads` 页面进一步简化为“筛选 + 结果 + 下一步”。
2. 对旧线索、低质线索、无时间线索给出显式原因。
3. 每条线索增加一键进入“补联系方式”或“记录无效”。
4. 减少详情页跳转成本，让用户在列表里就能完成初筛。
