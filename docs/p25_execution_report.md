# P25 执行报告：傻瓜式四步主流程

## 目标

把 Lead Radar 从“很多功能模块”收敛成普通用户也能直接理解的主流程：采集线索、筛选线索、补联系方式、跟进和复盘。日常使用只看这条链路，高级维护功能放到后面。

## 已完成

1. 首页改成 4 步获客流程
   - 第 1 步：采集线索
   - 第 2 步：筛选线索
   - 第 3 步：补联系方式
   - 第 4 步：跟进和复盘

2. 首页主按钮更智能
   - 如果平台有登录、验证、频控、低质输出问题，优先提示检查采集。
   - 如果已有高质量线索，优先进入筛选、补联系或跟进。

3. 左侧菜单收敛
   - 主菜单只保留日常流程和复盘入口。
   - 采集维护、质检、导入、效果分析、设置等放进“设置与高级工具”。
   - 菜单名改成动作型表达，减少术语感。

4. 顶部流程条统一
   - 核心页面统一显示 1/2/3/4 流程条。
   - 当前页面高亮，用户能一眼知道自己处在哪一步。

5. 首页标准动作说明
   - 明确写出每天应该按什么顺序推进。
   - 强调只保留 2026 年后的真实动态住宅 IP 需求。

## 修改文件

- `app/services/home_ops.py`
  - 首页数据模型从 3 步升级为 4 步。
  - 增加到期跟进数量。
  - 主动作选择逻辑改为优先处理平台阻塞，再进入下一步。

- `app/templates/home.html`
  - 首页文案和流程区改成 4 步。
  - “不要迷路”改为“标准动作”。

- `app/templates/base.html`
  - 左侧菜单重排。
  - 顶部流程条增加步骤编号。
  - 首页流程卡片支持 4 列。

## 当前判断

P25 后，工具已经不再只是“功能堆叠”，而是有一条清楚的日常作业路径。新用户打开首页后，可以直接按 1、2、3、4 做，不需要理解所有后台模块。

## 下一步建议

P26 应该继续做“实用性收口”：把每一步里的按钮再减少，把每个页面的主按钮固定成一个最推荐动作，其他操作放到二级菜单。目标是让用户不用想，照着系统提示每天推进 30 分钟也能产出可跟进线索。
